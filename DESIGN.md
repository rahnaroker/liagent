# Редактор электронных книг (FB2 → EPUB) — проектный документ

## 0. Резюме решений

| Параметр | Решение |
|---|---|
| Язык/стек | Go |
| GUI | Wails (frontend HTML/CSS/JS, backend Go) |
| Распространение | Один портабл-`.exe`, полностью офлайн |
| Источник | FB2 (часто windows-1251, из OCR/сканов) |
| Цель | EPUB3 для чтения на Kindle (Send to Kindle принимает EPUB) |
| Движок правок | Регулярные выражения + небольшие stateful-проходы, без LLM |
| Применение правок | Уровни **A и B — автоматически**; уровень **C — только ревью**; необратимые правила уровня B гейтятся словарём (см. §4.4) |
| Языки текста | Русский |

Принцип: **ядро на чистом Go (тестируемое, может работать как CLI) отделено от GUI.**
GUI лишь вызывает ядро.

```
fb2 ─►[Parser]─►[Document AST]─►[Rule Engine]─►[Findings]
                                      │
                            авто-применение A+B
                                      ▼
                   AST' ─►[EPUB Builder]─►[Validator]─► out.epub
                                      ▲
                       (GUI: пост-ревью диффа, откат правок)
```

Так как A+B применяются автоматически, GUI работает как **пост-ревью**: показывает полный
дифф, сгруппированный по правилам, с возможностью отката по правилу/экземпляру. Перед
правками всегда делается резервная копия исходника и сохраняется полный журнал изменений.

## 2. Парсинг FB2

### 2.1 Кодировка (критично)
FB2 часто в **windows-1251**. Алгоритм:
1. Прочитать первые байты, распарсить XML-декларацию `<?xml ... encoding="..."?>`.
2. Если кодировка ≠ UTF-8 — транскодировать через `golang.org/x/text/encoding`
   (charmap.Windows1251 и др.) в UTF-8, заменив декларацию.
3. Эвристическая страховка: если объявлен UTF-8, но встречаются невалидные
   последовательности — попробовать 1251 и сравнить долю кириллицы.
4. Зачистить управляющие символы и U+FFFD (битые символы плохой перекодировки).

### 2.2 AST
Парсим в типизированную модель (`core/model`), не зависящую от формата:

- **Description**: title-info (авторы: first/middle/last, book-title, annotation,
  keywords, date, lang, translator, sequence{name,number}, coverpage),
  document-info (id, version, program-used), publish-info (publisher, city, year, isbn).
- **Bodies**: основной body + дополнительные; отдельно `body[name="notes"]` (сноски).
- **Section** (рекурсивно): id, title, epigraph(s), image, annotation, content[].
- **Block-узлы**: P, Subtitle, EmptyLine, Poem{title,epigraph,stanza[],textAuthor},
  Stanza{title,subtitle,v[]}, Cite{...,textAuthor}, Epigraph, Annotation, Table{tr[]→{th/td}},
  Image(href, alt).
- **Inline-узлы**: Text, Strong, Emphasis, Style(name), Strikethrough, Sub, Sup, Code,
  Link(href, type), InlineImage.
- **Binaries**: id, content-type, base64-данные (декодируются лениво при сборке).

Парсер **толерантен к ошибкам** (битый FB2 — норма): не падать, восстанавливаться,
логировать проблемные места как findings структурного уровня.

## 3. FB2 → EPUB3: чистая конвертация с сохранением вёрстки

Это половина ценности проекта. Цель — чистый семантический XHTML, корректная вёрстка,
разбиение на главы, сохранение стилей, рабочая навигация и всплывающие сноски на Kindle.

### 3.1 Двухпроходная сборка
- **Проход 1 (планирование):** обойти AST, распределить секции по XHTML-файлам, построить
  карту `id → (файл, якорь)` для всех ссылочных элементов (секции, сноски, эпиграфы,
  изображения), собрать список бинарников и уникальных стилей.
- **Проход 2 (рендер):** сгенерировать XHTML, **резолвя внутренние ссылки** по карте,
  затем OPF/nav/ncx/CSS.

Разделение проходов обязательно: перекрёстные ссылки (`<a href="#id">`) и сноски нельзя
разрешить за один проход.

### 3.2 Разбиение на главы (нюанс)
Наивное «каждая секция = файл» ломается на вложенности (часть → главы). Стратегия:
- Файл создаётся для каждой **секции верхнего уровня** основного body.
- **Вложенные** титульные секции остаются в том же файле, но получают якоря и **пункты
  навигации** (иерархия сохраняется в nav/ncx).
- **Защита от гигантских файлов:** если файл превышает порог (~250 КБ), дробить глубже —
  Kindle плохо пагинирует большие файлы.
- Титул книги (body `<title>`/`<epigraph>`) → отдельный фронт-маттер файл.
- Обложка → отдельный `cover.xhtml` (`epub:type="cover"`).
- Сноски → отдельный `notes.xhtml`.

### 3.3 Маппинг узлов в XHTML
| FB2 | XHTML | Класс/семантика |
|---|---|---|
| section | `<section>` + файл/якорь | — |
| title (в секции) | `<h1>…<h6>` по глубине | строки `<p>` объединяются |
| subtitle | `<h4 class="subtitle">` | |
| p | `<p>` | text-indent + justify |
| empty-line | `<p class="empty-line"> </p>` | вертикальный отступ, **серии схлопываются** |
| epigraph | `<div class="epigraph">` | курсив, сдвиг вправо |
| cite | `<blockquote class="cite">` | отступ |
| text-author | `<p class="text-author">` | вправо, курсив |
| poem | `<div class="poem">` | сохранять разбивку строк |
| stanza / v | `<div class="stanza">` / `<p class="v">` | каждая строка стиха = блок |
| annotation | `<div class="annotation">` | |
| table/tr/th/td | `<table>`… | align/colspan/rowspan переносятся |
| image (блочн.) | `<div class="image"><img/></div>` | alt из FB2 |
| strong/emphasis | `<strong>`/`<em>` | семантика сохраняется |
| style[name] | `<span class="st-{name}">` | класс на каждый уникальный стиль |
| strikethrough/sub/sup/code | `<s>`/`<sub>`/`<sup>`/`<code>` | |
| a[type=note] | noteref (см. 3.5) | |

### 3.4 Изображения и обложка
- `<binary>` (base64) → файлы в `OEBPS/images/`, расширение по content-type.
- Дедупликация id; санитизация имён.
- Обложка: в OPF `<meta name="cover" content="cover-img"/>` (Kindle требует именно так)
  **и** `cover.xhtml` в начале spine.
- Инлайн-изображения: `<img class="inline"/>`, `max-width:100%`.

### 3.5 Сноски → всплывающие на Kindle
- Ссылка в тексте: `<a epub:type="noteref" href="notes.xhtml#note_1">[1]</a>`.
- Цель в notes.xhtml: `<aside epub:type="footnote" id="note_1">…</aside>`.
- Kindle (KF8) при такой разметке показывает **всплывающие сноски** — важный плюс к качеству.
- Битые ссылки (noteref без цели) → finding структурного уровня.

### 3.6 Метаданные (content.opf)
- `dc:title`, `dc:creator` (role=aut, `file-as="Фамилия, Имя"` для сортировки),
  `dc:language=ru`, `dc:identifier` (FB2 id как UUID, иначе сгенерировать),
  `dc:date`, `dc:publisher`, `dc:description` (annotation), `dc:source` (isbn),
  переводчик role=trl.
- Серия: EPUB3 `belongs-to-collection` + `<meta name="calibre:series">` (совместимость).
- Обязательное EPUB3 `dcterms:modified`.
- `spine`: cover → titlepage → главы… → notes.
- `manifest` со свойствами (`nav`, `cover-image`).
- `guide` (EPUB2) для cover/toc/text — помогает старым Kindle.

### 3.7 Навигация
- `nav.xhtml` (EPUB3) — иерархия из вложенных title секций.
- `toc.ncx` (EPUB2) — для совместимости со старыми движками Kindle.
- Заголовки извлекаются из `<title>` (конкатенация `<p>`-строк).

### 3.8 CSS (минимальный, Kindle-safe)
Только то, что Kindle уважает; единицы — `em`:
- `body` поля; `p` text-indent + text-align: justify;
- `h1..h6`; `.subtitle`, `.epigraph`, `.cite`, `.text-author`, `.poem`, `.stanza`, `.v`,
  `.annotation`, `.empty-line`, `.image img{max-width:100%}`, `.cover`.
- **Не** вставлять мягкие переносы / принудительный перенос — Kindle переносит сам.
- Не полагаться на @font-face и сложные поля (Kindle часто игнорирует).

### 3.9 Чистота XHTML
- Well-formed XHTML5: `<!DOCTYPE html>`, `xmlns`, `xmlns:epub`, корректное экранирование.
- id санитизируются в валидный NCName (FB2 id может начинаться с цифры → префикс).
- Пустые `<p></p>` не эмитятся.
- Нормализация пробелов, но **сохранение разбивки строк стиха**.
- Внутренние перекрёстные ссылки резолвятся в `файл#якорь`.

### 3.10 Выходной комплект EPUB
```
mimetype (несжатый, первым)
META-INF/container.xml
OEBPS/content.opf
OEBPS/nav.xhtml
OEBPS/toc.ncx
OEBPS/style.css
OEBPS/cover.xhtml
OEBPS/text/chap_0001.xhtml …
OEBPS/notes.xhtml
OEBPS/images/…
```

### 3.11 Валидация
- Внутренние проверки: битые ссылки, отсутствующие картинки, корректность OPF/manifest/spine,
  валидность XHTML.
- Полноценный epubcheck (Java) офлайн-портабл не тащим; рекомендуем в README как внешний шаг.

## 4. Движок правил (регулярки)

### 4.1 Модель правила
```go
type Confidence int // A=safe, B=heuristic, C=review-only
type Rule struct {
    ID, Name, Category string
    Confidence  Confidence
    Re          *regexp.Regexp     // или nil для stateful/функциональных правил
    Apply       func(string) string
    AutoDefault bool
    Scope       Scope              // TextNode | Paragraph | Markup
    Guard       func(ctx) bool     // напр. наличие словаря для необратимых
}
```
- Регулярки компилируются один раз. Сканирование секций — параллельно (goroutine).
- RE2 линеен → быстро даже на больших книгах.
- Каждое срабатывание → **finding** {секция, offset, контекст ±N, before, after, ruleID, level}.

### 4.2 Порядок выполнения (важен)
1. Удалить soft hyphen (U+00AD), управляющие/битые символы.
2. Склейка слов, разорванных переносом (гейт словарём, см. 4.4).
3. Схлопнуть пробелы / убрать хвостовые.
4. Кавычки (stateful-проход, см. 4.4).
5. Тире, многоточие, апострофы.
6. Неразрывные пробелы (предлоги, инициалы, сокращения, перед тире).
7. Структурные нормализации (серии empty-line, разделители сцен).

### 4.3 Защищённые зоны (исключаются из текстовых правил)
- `<code>`, URL/email, числа (для правила «пробел после точки»),
- **словарь сокращений** (`т. е.`, `и т. д.`, `и т. п.`, `и др.`, `Ф. И. О.`, `н. э.` …) —
  чтобы не разбивать «точка+заглавная» и не ломать сокращения.

### 4.4 Особые (не-наивные) проходы
- **Кавычки** — не наивная регулярка, а **stateful-проход** по абзацу: отслеживание
  состояния открыто/закрыто, внешние «ёлочки», вложенные „лапки". Надёжнее regex на
  смежных/вложенных кавычках.
- **Склейка переносов** `сло-\nво`/`сло- во` — **гейтится словарём**: склеиваем
  автоматически, только если итоговое слово есть в частотном словаре; иначе — finding
  уровня C (ревью). Без словаря правило не применяется молча (необратимый риск).
- **Смешанный алфавит** (латиница в кириллице) — авто-нормализация **только** если слово
  целиком кириллическое за вычетом 1–2 латинских двойников (`с/c`, `о/o`, `е/e`, `а/a`,
  `р/p`, `х/x`, `Н/H`, `В/B`, …); иначе ревью.
- **Восстановление «ё»** — словарь, **всегда review-only** (уровень C), низкая уверенность.

> Так как A+B авто, риск-правила (склейка, смешанный алфавит, ё) защищены гейтом/словарём
> или понижены до C. Полный журнал + бэкап делают любые авто-правки откатываемыми.

### 4.5 Каталог правил

#### Уровень A — авто, безопасно
| ID | Ошибка | Действие |
|---|---|---|
| sp-multi | Множественные пробелы | `\s{2,}` → ` ` |
| sp-before-punct | Пробел перед `,.;:!?…»)` | убрать |
| sp-after-punct | Нет пробела после `,.;:!?` | добавить (с защитой 4.3) |
| sp-paren | Пробел внутри `( … )` | убрать внутренние |
| sp-trim | Хвостовые/ведущие пробелы абзаца | trim |
| sp-newline | Жёсткий перенос внутри абзаца | → пробел |
| dash-hyphen | Дефис как тире ` - ` | → ` — ` |
| dash-multi | `--`, `---` | → `—` |
| dash-range | Диапазон чисел `1990-2000` | → `–` (en dash) |
| ell-dots | `...` | → `…` |
| soft-hyphen | U+00AD | удалить |
| sym-copy | `(c)(tm)(r)` | → `© ™ ®` |
| punct-dup | `,,` `. .` (не `…`,`!!!`) | схлопнуть |
| ctrl-clean | Управляющие/U+FFFD | удалить |

#### Уровень B — авто, эвристика
| ID | Ошибка | Подход |
|---|---|---|
| quotes | Прямые `"` → «…» / вложенные „…" | stateful-проход |
| quotes-apos | `'` между букв → `’` | `(\p{L})'(\p{L})` |
| dialog-dash | `- Реплика` в начале абзаца → `— ` | начало + пробел |
| nbsp-prep | Неразр. пробел после предлогов/союзов (`в на с и к о у…`) | список + ` ` |
| nbsp-init | Инициалы `А. С. Пушкин` | паттерн инициалов |
| nbsp-abbr | Сокращения `и т. д.` | список сокращений |
| nbsp-num | Число + единица/`%`/`№`/`г.` | ` ` |
| nbsp-dash | Неразр. пробел **перед** тире | `слово — ` |
| ell-quest | `...?` → `?..`, `...!` → `!..` | русская норма |
| dehyph | Склейка переносов | **гейт словарём** (4.4) |
| mixed-alpha | Латиница в кириллице | условная (4.4) |
| ocr-digit | Цифра-двойник в слове `б6 О0 1l` | внутри слова |
| glued-dot | `конец.Начало` | `([а-яё])\.([А-ЯЁ])`→`$1. $2` (защита сокр.) |
| scene-break | `***` `* * *` | нормализовать разделитель |

#### Уровень C — только ревью
| ID | Ошибка | Почему |
|---|---|---|
| yo-restore | Восстановление «ё» | неоднозначно, нужен словарь |
| quote-punct | Точка/запятая относительно `»` | сложные исключения (?!, …) |
| heading-detect | Абзац выглядит как заголовок (короткий, ЦЕНТР, ЗАГЛАВНЫЕ, без точки) | риск ложного |
| page-number | Одинокая цифра в абзаце (номер страницы OCR) | может быть легит. |
| running-header | Повтор колонтитула между секциями | эвристика |
| glued-words | Слипшиеся слова `словоСлово` | нужен словарь |
| spaced-emphasis | Разрядка `с л о в о` | шумно |
| empty-para | Пустой/пробельный `<p>` | контекст вёрстки |
| broken-noteref | Сноска без цели | требует решения |

### 4.6 Безопасность правок
- Перед запуском — **бэкап исходного FB2**.
- **Полный журнал** правок (ruleID, позиция, before/after) — экспорт в отчёт.
- В GUI — пост-ревью диффа с откатом по правилу и по экземпляру.
- Идемпотентность: повторный прогон не должен ничего менять.

## 5. GUI (Wails)

- **Слева:** дерево findings по категориям/правилам со счётчиками; тумблеры правил;
  «откатить всё по правилу».
- **Центр:** превью главы с подсветкой изменённых спанов; инлайн-дифф before/after.
- **Справа:** карточка правила (описание, уровень A/B/C), откат экземпляра.
- **Верх:** открыть FB2 / батч-список; настройки (агрессивность nbsp, словари вкл/выкл,
  какие уровни авто-применять, порог дробления глав); кнопка «Собрать EPUB».
- **Вкладка:** предпросмотр итоговой главы EPUB.

Производительность: компиляция регексов один раз, скан по секциям в goroutine,
RE2 линеен — крупная книга обрабатывается за доли секунды.

## 6. Roadmap (кодим по шагам)

1. **Скелет:** go mod, Wails init, каталоги, пустое окно + «открыть файл».
2. **Парсер FB2** + транскодинг 1251/UTF-8 → AST. Golden-тесты на testdata.
3. **EPUB-билдер** (без правок): FB2→валидный EPUB3 (главы, nav/ncx, обложка, сноски,
   CSS). Вертикальный срез — уже полезно. Проверка в ридере/epubcheck.
4. **Движок правил, уровень A** + юнит-тесты на каждое правило.
5. **Findings + GUI пост-ревью:** список, подсветка, дифф, откат, бэкап, журнал.
6. **Уровень B** (кавычки stateful, реплики, nbsp, склейка с гейтом, смешанный алфавит).
7. **Уровень C** (ревью-правила) + словари (ё, частотный список).
8. **Доводка качества Kindle:** всплывающие сноски, дробление крупных файлов, метаданные.
9. **Батч-режим, настройки, сборка портабл-`.exe`.**

## 7. Риски и подводные камни
- **Кодировка FB2 (1251)** — ловить на шаге 2, иначе кракозябры.
- **Дизамбигуация кавычек** — только stateful, не наивная regex.
- **Склейка переносов без словаря** — необратимый риск, гейтить словарём.
- **Вложенность секций** при дроблении на главы — не терять иерархию nav.
- **Перекрёстные ссылки/сноски** — обязательно двухпроходная сборка.
- **Битый/нестандартный FB2** — толерантный парсер, не падать.
